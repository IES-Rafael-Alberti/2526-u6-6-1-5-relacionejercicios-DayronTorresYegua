# Ejercicio 6.1 — Tipos de herencia, clases y subclases (RA7.a)

Basado en la teoría de jerarquía de clases (Unidad 6.1).

## Objetivo

Crear **dos** estructuras de herencia (elige **2** de estas 4): **Especialización**, **Extensión**, **Especificación**, **Construcción**.

> Nota: el comportamiento no tiene que ser “realista”, pero las clases deben diferenciarse claramente para que se entienda el **tipo de herencia**.

I.a) Para **cada una** de las 2 estructuras elegidas:
   - Piensa en **(1) una superclase** y **(2) dos subclases**.

RESPUESTA: Explica tu propuesta de superclase y subclases para cada estructura, y el tipo de herencia que representa.

**Primera propuesta:** He usado una estructura de especializacion

La superclase es Vehiculo, que representa un vehículo en general. Tiene atributos como marca, modelo, capacidad del tanque y combustible actual. También tiene un método para llenar el tanque.

Las subclases son:
- Coche
- Motocicleta

Estas clases heredan de Vehiculo, pero cada una tiene comportamientos propios. Por ejemplo:

- El coche puede hacer un derrape
- La motocicleta puede hacer un caballito

Esto representa una especialización porque ambas clases son tipos más específicos de vehículo.

**Segunda propuesta:** He usado una esctructura de extension

La superclase es Autenticador. Esta clase tiene un método llamado controlarAutenticacion, que simplemente indica que el usuario ha sido autenticado.

Las subclases son:
- AutenticadorPin
- AutenticadorPatron

Estas clases amplían el comportamiento de la superclase añadiendo más pasos antes de autenticar al usuario. 
Por ejemplo:

- AutenticadorPin verifica un pin
- AutenticadorPatron verifica un patron

Después de verificar, se llama al método de la superclase.

## II. Teoría (documentar aquí)

Para **cada** estructura elegida, responde en este Markdown (Usa Enlaces permanentes a código para ilustrar tus respuestas):

II.a) ¿Qué tipo de herencia es? (Especialización, Extensión, Especificación, Construcción).  

RESPUESTA:

**Primera propuesta:**

Es una herencia de tipo Especialización, porque Coche y Motocicleta son tipos más específicos de Vehiculo.

**Segunda propuesta:**

Es una herencia de tipo Extensión, porque las subclases añaden comportamiento nuevo al proceso de autenticación.

II.b) ¿Quién es la superclase y cuáles son las subclases?

RESPUESTA:

**Primera propuesta:**

Superclase: Vehiculo

Subclases: 
- Coche
- Motocicleta

**Segunda propuesta:**

Superclase: Autenticador

Subclases: 
- AutenticadorPin
- AutenticadorPatron

II.c) ¿Existe otra forma de implementarlo que no sea herencia? (composición, interfaces, delegación, etc.)

RESPUESTA:

**Primera propuesta:**

Sí, se podría usar composición. Por ejemplo, un vehículo podría tener un objeto que represente el tipo de comportamiento que puede hacer, en lugar de crear subclases.

**Segunda propuesta:**

También se podría usar interfaces. Por ejemplo, una interfaz MetodoAutenticacion que tenga diferentes implementaciones como PIN o patrón.

II.d) ¿Qué ventajas aporta esta implementación con respecto a la alternativa?

RESPUESTA:

**Primera propuesta:**

La herencia permite reutilizar código de la clase Vehiculo. Así no hay que repetir atributos como marca o combustible en todas las clases.

También hace que el código sea más fácil de entender porque se ve claramente la relación entre las clases.

**Segunda propuesta:**

Permite reutilizar el método de autenticación base y añadir pasos extra en cada subclase.

Además el código queda más organizado porque cada tipo de autenticación está en su propia clase.
